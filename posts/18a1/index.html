图形学-光线追踪(Whitted风格光线追踪)

为什么需要光线追踪？

光栅化很难表现一些全局效果，如阴影，光线多次反射等。

光线追踪可以表现的效果很好，但是渲染会非常慢。

图中的场景使用了 1w 核左右的 CPU 渲染了 1 个小时仅渲染出一帧的画面。

光线定义

  1. 假设光线沿直线传播
  2. 光线之间不会膨胀，它们交叉会直接穿过
  3. 光路可逆，从光源出发的光线传播到物体表面反射到相机中的光路，和从相机出发的光线传播到物体表面反射到光源中的光路是一样的

Ray Casting 光线投射

1968 年，Arthur Appel 提出了光线投射的概念。

  • 从相机出发，沿着每个像素的光线方向投射光线
  • 将交点与光源进行比较，判断光线是否被遮挡

但实际上光线是可以弹射多次的，下图为 Whitted-Style Ray Tracing 的光线追踪方法。

  • 从相机出发，沿着每个像素的光线方向投射光线，这个光线称作 primary ray
  • 光线与第一个物体产生交点，将交点与光源连线判定可见性，着色点与光源连线的光线称作 shadow ray
  • 如果可见，则该着色点将发生镜面反射和折射，反射折射的光线称作 secondary ray
  • 将所有 secondary ray 和 primary ray 到达的可见着色点接收到的光源进行累加，得到最终的像素颜色

线与面的交点计算

光线方程

光线由一个起点和一个方向向量确定。

r(t)=o+tdr(t) = o + td r(t)=o+td

其中，ooo为光线的起点，ddd为光线的方向向量，t≥0t \geq 0t≥0。

光线与球的交点

球由一个中心点和一个半径确定。

∥p−c∥2=R2\|p - c\|^2 = R^2 ∥p−c∥2=R2

与光线方程联立得：

(o+td−c)2−R2=0(o + td - c)^2 - R^2 = 0 (o+td−c)2−R2=0

将其展开得：

t2(d⋅d)+2(d⋅(o−c))t+((o−c)⋅(o−c)−R2)=0t^2(d \cdot d) + 2(d \cdot (o - c))t + ((o - c) \cdot (o - c) - R^2) = 0 t2(d⋅d)+2(d⋅(o−c))t+((o−c)⋅(o−c)−R2)=0

这是一个二次方程，使用求根公式可以得到：

t=−b±b2−4ac2at = \frac{-b \pm \sqrt{b^2 - 4ac}}{2a} t=2a−b±b2−4ac​​

其中，a=d⋅da = d \cdot da=d⋅d，b=2(d⋅(o−c))b = 2(d \cdot (o - c))b=2(d⋅(o−c))，c=(o−c)⋅(o−c)−R2c = (o - c) \cdot (o - c) - R^2c=(o−c)⋅(o−c)−R2。

  • 如果 b2−4ac<0b^2 - 4ac < 0b2−4ac<0，则光线与球没有交点。
  • 如果 b2−4ac=0b^2 - 4ac = 0b2−4ac=0，则光线与球有一个交点。
  • 如果 b2−4ac>0b^2 - 4ac > 0b2−4ac>0，则光线与球有两个交点。

光线与任意隐式表面的交点

定义隐式表面 p 上的任意一点 p 满足：

f(p)=0f(p) = 0 f(p)=0

与光线方程联立得：

f(o+td)=0f(o + td) = 0 f(o+td)=0

求解该方程得出 t 即可求出交点，t 需要确保是实数且非负。

光线与三角形面的交点

作用：

  • 可见性判定，阴影，光线计算
  • 判断某个点在几何体内部还是外部

平面方程

平面由平面上的一个点p′p'p′和一个平面的法向量NNN确定。

对于平面上的任意点ppp，满足：

N⋅(p−p′)=0N \cdot (p - p') = 0 N⋅(p−p′)=0

则所有这样的点ppp的集合就构成了一个平面。

光线与平面的交点计算

与光线方程联立，可得：

N⋅(o+td−p′)=0N \cdot (o + td - p') = 0 N⋅(o+td−p′)=0

解得：
t=N⋅(p′−o)N⋅dt = \frac{N \cdot (p' - o)}{N \cdot d}t=N⋅dN⋅(p′−o)​
最后检查t≥0t \geq 0t≥0即可。

光线与三角形面的交点计算

求出光线与平面的交点后，需要判断该交点是否在三角形内。

如果交点在三角形内，则该交点为光线与三角形面的交点。

还有一种算法是使用重心坐标系来判断光线与三角形面的交点。

由于三角形内的任意一点都可以使用三角形的重心坐标和三个顶点的线性组合来表示，因此可以将光线与三角形面的交点表示为：

O+tD=(1−b1−b2)P0+b1P1+b2P2O+tD = (1 - b_1 - b_2) P_0 + b_1 P_1 + b_2 P_2 O+tD=(1−b1​−b2​)P0​+b1​P1​+b2​P2​

最后可以同时求解出t,b1,b2t,b_1, b_2t,b1​,b2​，并检查b1,b2≥0b_1, b_2 \geq 0b1​,b2​≥0且b1+b2≤1b_1 + b_2 \leq 1b1​+b2​≤1即可。

光线追踪计算优化

包围盒/包围体 Bounding Box/Bounding Volume

包围盒是一个简单的几何体，用来包裹一个复杂的几何体。

如果一根光线连包围盒都碰不到，则光线与包围盒内的几何体也不会有交点。这样直接就可以裁剪掉大量不必要的计算。

我们通常使用一个轴对齐(Axis-Aligned Bounding Box, AABB)的包围盒来包裹一个几何体。

2D 包围盒

光线从 o 点出发，方向为ddd，

在 x 方向上，可以求出光线与第一条直线在tmint_{min}tmin​时从x0x_0x0​进入，在tmaxt_{max}tmax​时从x1x_1x1​射出。

在 y 方向上，可以求出光线与第二条直线在tmint_{min}tmin​时从y0y_0y0​进入，在tmaxt_{max}tmax​时从y1y_1y1​射出。

最后在两个方向上的 t 值取交集即可得到最终光线在包围盒内的传播时间。

这个求交集的过程可以转化成一个逻辑问题：

  • 设tentert_{enter}tenter​为光线的进入时间，则光线必须至少穿过所有轴向的边界线才能算作光线进入包围盒，即对于每个轴向都可以算出一个光线进入时间tmint_{min}tmin​，取所有轴向的最大值作为tentert_{enter}tenter​。即tenter=max⁡tmint_{enter} = \max{t_{min}}tenter​=maxtmin​。

  • 设texitt_{exit}texit​为光线的退出时间，则光线必须至少穿过任意一个轴向的边界线才能算作光线退出包围盒，即对于每个轴向都可以算出一个光线退出时间tmaxt_{max}tmax​，取所有轴向的最小值作为texitt_{exit}texit​。即texit=min⁡tmaxt_{exit} = \min{t_{max}}texit​=mintmax​。

3D 包围盒

可以沿用上述 2D 包围盒的逻辑来求解 3D 包围盒。取tenter=max⁡tmint_{enter} = \max{t_{min}}tenter​=maxtmin​, texit=min⁡tmaxt_{exit} = \min{t_{max}}texit​=mintmax​作为光线在包围盒内的传播时间。

但是注意，光线是一个射线而不是一个直线，所以应当需要检查ttt是否有意义。

  • 当 $t*{enter} < t*{exit} < 0 $时，盒子在光线的背后时，光线在盒子外部，与盒子一个交点都没有，需要排除。
  • 当 tenter<0<=texitt_{enter} < 0 <= t_{exit}tenter​<0<=texit​时，光线在盒子内部出发，与盒子有一个交点射出到外面。
  • 当 0<=tenter<texit0 <= t_{enter} < t_{exit}0<=tenter​<texit​时，光线在盒子外部出发，与盒子有一个交点射入到里面，从另一个交点射出到外面。

故光线与 AABB 盒存在交点的判定条件可以总结如下：

tenter<texit∧texit≥0t_{enter} < t_{exit} \land t_{exit} \geq 0 tenter​<texit​∧texit​≥0

Uniform grids 均匀网格划分

先进行场景预处理，

  • 计算出物体的包围盒
  • 创建一个均匀划分的网格
  • 在物体所覆盖的每个网格单元中存储该物体

将场景划分为均匀的网格，光线投射经过的网格与物体所覆盖的网格进行求交集，若存在交集，则有可能有交点。

若格子划分的太稀疏，则每个格子中可能会有很多物体，光线投射时需要遍历该格子中的所有物体，效率低下。

若格子划分的太密集，则需要做很多次光线投射，效率也会降低。

人们找到了一个经验公式是综合来看最高效的网格数选取方案，即cells=C∗objscells = C * objscells=C∗objs，其中CCC为一个常数，objsobjsobjs为物体的数量。在三维空间中，CCC通常取值为 27。

Spatial partitions 空间划分

之前的均匀网格划分的弊端就是当场景中的物体分布不均匀时，可能导致很大一块空白区域划分了很多网格，而这些网格中并没有任何物体。也有可能某些区域物体过于密集，导致每个格子中包含了大量物体。都会影响光线追踪的计算效率。故需要使用空间划分的方法来进一步优化。

  • Oct-Tree 八叉树
    • 基本算法：
      • 将空间每次递归地均分为八个子空间
      • 如果一个子空间中包含的物体数量超过某个阈值，则继续划分该子空间
      • 直到达到最大划分深度或者所有叶子空间中的物体数量低于阈值
    • 弊端：
      • n 维空间划分时，子空间数量呈指数增长，导致存储开销过大
  • KD-Tree KD 树
    • 基本算法：
      • 将空间递归地划分为两个子空间
      • 依次按照 x, y, z 轴向的平面进行划分，划分界限不一定是中点，可以自行选取
      • 如果一个子空间中包含的物体数量超过某个阈值，则继续划分该子空间
      • 直到达到最大划分深度或者所有叶子空间中的物体数量低于阈值
    • 优点：
      • 相比八叉树，KD 树在高维空间中更节省存储空间，只会是一个二叉树
  • BSP-Tree BSP 树
    • 基本算法：
      • 将空间递归地划分为两个子空间
      • 划分平面可以是任意方向的平面，可以根据场景中的物体分布情况来选取划分平面
      • 如果一个子空间中包含的物体数量超过某个阈值，则继续划分该子空间
      • 直到达到最大划分深度或者所有叶子空间中的物体数量低于阈值
    • 优点：
      • 相比 KD 树，BSP 树可以更灵活地适应场景中的物体分布情况，通常能得到更好的划分效果
      • 但由于划分平面不再是轴对齐的，故光线与 BSP 树的求交运算会更复杂一些

Object Partitions & Bounding Volume Hierarchies (BVH)

上述都是按空间的角度进行划分的，而 BVH 则是按物体的角度进行划分。

BVH 算法的基本流程：

  • 计算包围盒
  • 递归地将所有物体划分为两个子集
  • 重新分别计算两个子集内的包围盒
  • 如果一个子集内的物体数量超过某个阈值，则继续划分该子集
  • 直到达到最大划分深度或者所有叶子空间中的物体数量低于阈值
  • 每个叶子节点中存储一个包围盒和该包围盒内的所有物体

如何构建一个 BVH 树？

  • 类似 KD-Tree, 每次分别沿着 x, y, z 轴向进行划分
  • 总是按分布最长的轴向进行划分
  • 每次取中间位置的物体进行划分
    • 如果每次取中间物体进行划分，则树的两侧会比较平衡，树的深度可以最小，查询效率最高
    • 一个无序的物体列表，取中间物体进行划分，可以类似快速排序的思想，使用快速选择算法，时间复杂度为 O(n)，也可以直接使用快速排序取中间，时间复杂度为 O(n log n)

BVH 树的数据结构：

  • 内部节点
    • 包围盒
    • children 指针
  • 叶子节点
    • 包围盒
    • 物体列表
